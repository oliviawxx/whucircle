WHU Circle 需求建模文档

一、用户需求说明

1.1 项目概述

WHU Circle 是面向武汉大学学生的轻量级社交媒体原型系统，采用前后端分离的单页 Web 应用架构。系统以动态分享、兴趣频道、私信沟通、个人主页和隐私控制为核心，支持校内邮箱注册、图文笔记发布、关注与社交圈、频道讨论、即时通讯等校园社交场景。

1.5 核心业务需求

用户认证：仅 @whu.edu.cn 邮箱可注册。验证码 5 分钟有效、60 秒重发冷却、最多连续输入 5 次错误。密码使用 BCrypt 哈希存储。登录后服务端签发 Bearer Token，客户端在请求头携带。除 /auth/* 和 /health 外，所有 API 均需 Token 鉴权，无效 Token 返回 40100。

笔记系统：用户可发布笔记，包含标题（≤100 字）、正文（必填）、图片、标签。可见性分 PUBLIC（公开）、FRIENDS（好友可见）、PRIVATE（仅自己）。首页广场展示公开笔记，支持关键词搜索和标签筛选，分页返回（每页 20 条）。笔记详情按可见性校验：好友可见需互关关系（FRIEND），私密仅作者本人。点赞和评论支持切换状态与计数更新。评论 ≤1000 字，被拉黑用户不可评论。仅作者可删除笔记和评论。收藏笔记可在"我的收藏"中查看。

社交关系：关注为单向操作。若对方已关注自己，关系自动升级为 FRIEND（互关好友）。取关后好友关系自动降级。社交圈（/feed/social）仅展示已关注用户的笔记，好友可见内容需互关才显示。拉黑时自动取关，双向屏蔽主页访问。关系状态枚举：NONE → FOLLOWING → FOLLOWER → FRIEND → BLOCKED。

频道系统：支持 PUBLIC（公开加入）和 PASSWORD（密码验证）两种加入方式。创建者自动成为管理员和首位成员。仅成员可发帖、回复（≤1000 字）和点赞。未加入频道时仅预览最近 5 条帖子。仅管理员可修改频道公告和置顶/取消置顶帖子。

即时通讯：支持 PRIVATE 私聊和 GROUP 群聊。创建私聊会话时需校验黑名单关系和对方私信权限（EVERYONE / FRIENDS_ONLY / NONE）。消息 ≤2000 字，当前采用 HTTP 请求模式（非 WebSocket）。支持消息已读标记。会话列表按最后消息时间倒序，显示未读消息数。

通知与安全：笔记被点赞、评论、收藏以及频道帖子被回复时，自动为内容作者生成未读通知。支持通知分页列表、单条已读、全部已读、未读数角标。隐私设置三项：默认笔记可见性、默认频道加入方式、私信权限。举报目标类型：NOTE（笔记）、CHANNEL_POST（频道帖子）、MESSAGE（消息）、USER（用户）。举报状态流转：PENDING → RESOLVED / REJECTED。

四、拓展实现目标

  • 活动组织与事件日历：支持活动发布、报名、提醒和日历视图
  • 平台管理员后台：举报审核、违规处理、基础数据统计
  • 推荐与发现优化：基于标签和互动行为的内容排序
  • WebSocket 实时聊天：将当前 HTTP 聊天升级为实时消息推送
  • 文件存储服务：图片外链替换为服务器存储或对象存储
  • 统一搜索：支持用户、笔记、频道、频道帖子的全局搜索
  • 移动端适配：优化手机浏览器下的布局和操作体验
  • 站内智能助手：接入 Agent，支持自然语言查询校园信息

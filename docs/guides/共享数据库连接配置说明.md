# WHU Circle 共享 MySQL 数据库连接配置说明

本文档给队友使用，用来把本机后端连接到组长电脑上正在运行的共享 MySQL 数据库。

当前方案是：每个人在自己的电脑上启动前端和后端，但后端统一连接组长电脑通过 Tailscale 暴露出来的 MySQL。这样大家看到的是同一份用户、帖子、评论、频道、通知等业务数据。

## 0. 当前连接信息

| 项目 | 值 |
| --- | --- |
| 数据库主机 | `100.97.230.36` |
| 数据库端口 | `3306` |
| 数据库名 | `whu_circle` |
| 数据库账号 | `whu_team` |
| 数据库密码 | 向组长单独索要，不要写进 GitHub、截图或群公告 |
| 后端 profile | `mysql` |

当前数据库账号权限是：

```sql
GRANT SELECT, INSERT, UPDATE, DELETE ON whu_circle.* TO 'whu_team'@'100.%';
```

这表示队友可以读取、新增、修改、删除 `whu_circle` 业务库里的数据，足够测试注册、登录、发帖、评论、点赞、关注、拉黑、通知、聊天等功能。它没有 `DROP`、`ALTER`、`CREATE`、`GRANT OPTION` 等权限，所以不能删除整个数据库、改表结构或把权限继续分配给别人。

依据：
- MySQL `GRANT` 支持在 `db_name.*` 粒度授予权限；官方语法里 `priv_level` 包含 `db_name.*`。
- MySQL 官方说明 `WITH GRANT OPTION` 会让账号把自己拥有的权限再授权给别人，所以本文档不授予它。
- 参考：MySQL GRANT Statement: https://dev.mysql.com/doc/refman/8.4/en/grant.html

## 1. 整体连接关系

```text
队友浏览器
  -> 队友本机 Vite 前端 http://127.0.0.1:5173
  -> 队友本机 Spring Boot 后端 http://127.0.0.1:8080
  -> Tailscale 私有网络
  -> 组长电脑 MySQL 100.97.230.36:3306
```

也就是说，队友不需要把自己的 MySQL 配出来；只要自己的后端能连上 `100.97.230.36:3306`，项目就能使用共享数据库。

依据：
- 项目后端 `D:\whu-circle-prototype\backend\src\main\resources\application-mysql.yml` 使用 `DB_URL`、`DB_USERNAME`、`DB_PASSWORD` 连接 MySQL。
- Spring Boot 官方说明支持从环境变量读取配置，并支持使用 `${name:default}` 形式的占位符和默认值。
- Tailscale 官方说明每台设备会得到稳定的 `100.x.y.z` 地址，适合在私有 tailnet 内互相访问。
- 参考：Spring Boot Externalized Configuration: https://docs.spring.io/spring-boot/reference/features/external-config.html
- 参考：How Tailscale assigns IP addresses: https://tailscale.com/docs/concepts/ip-and-dns-addresses

## 2. 队友第一步：安装并登录 Tailscale

1. 打开浏览器，进入 Tailscale Windows 官方安装页面：

   https://tailscale.com/docs/install/windows

2. 点击页面里的下载入口，下载安装包。

3. 双击安装包并完成安装。

4. 安装完成后，Windows 右下角系统托盘会出现 Tailscale 图标。如果没看到，点一下托盘里的向上箭头。

5. 右键 Tailscale 图标，选择登录。

6. 用组长邀请加入的账号完成浏览器登录。

为什么要这样做：
- 共享数据库没有直接开放到公网，而是只允许 Tailscale 私有网络里的设备访问。
- 不登录 Tailscale 时，电脑没有 tailnet 内的 `100.x.y.z` 地址，也就访问不到组长电脑的 MySQL。

依据：
- Tailscale 官方 Windows 安装文档说明：下载安装程序、运行安装、系统托盘出现图标、右键图标并登录。
- 参考：Install Tailscale on Windows: https://tailscale.com/docs/install/windows

## 3. 组长需要先邀请队友加入 tailnet

这一节给组长操作。

1. 打开浏览器，进入 Tailscale 管理后台：

   https://login.tailscale.com/admin/users

2. 登录组长的 Tailscale 账号。

3. 在用户管理页面里邀请队友账号加入当前 tailnet。

4. 队友接受邀请后，让队友完成第 2 节的安装和登录。

为什么要这样做：
- Tailscale 只有同一个 tailnet 内的设备才能用私有地址互相访问。
- 组长电脑的 MySQL 防火墙规则只允许 Tailscale 私有地址段访问，不面向公网。

依据：
- Tailscale 官方有邀请用户加入 tailnet 的说明。
- 参考：Invite any user to your tailnet: https://tailscale.com/docs/features/sharing/how-to/invite-any-user

## 4. 队友第二步：检查是否能看到组长电脑

打开 PowerShell，输入：

```powershell
tailscale ping 100.97.230.36
```

如果命令显示能 ping 通，说明 Tailscale 私有网络连通。

如果提示找不到 `tailscale` 命令：

1. 确认 Tailscale 已安装并正在运行。
2. 重新打开一个 PowerShell。
3. 或者使用完整路径：

```powershell
& "C:\Program Files\Tailscale\tailscale.exe" ping 100.97.230.36
```

为什么要这样做：
- 这一步只检查 Tailscale 网络，不检查 MySQL。
- 如果这里不通，后面的数据库连接一定也不会通。

依据：
- Tailscale 官方说明设备会获得稳定的 Tailscale IP；本项目使用组长电脑的 `100.97.230.36` 作为数据库入口。
- 参考：How Tailscale assigns IP addresses: https://tailscale.com/docs/concepts/ip-and-dns-addresses

## 5. 队友第三步：检查数据库端口是否能连通

继续在 PowerShell 输入：

```powershell
Test-NetConnection 100.97.230.36 -Port 3306
```

重点看输出里的：

```text
TcpTestSucceeded : True
```

如果是 `True`，说明队友电脑能连到组长电脑的 MySQL 端口。

如果是 `False`，常见原因是：
- 队友还没有登录 Tailscale。
- 队友没有加入组长的 tailnet。
- 组长电脑关机、休眠，或 MySQL 服务停止。
- 组长电脑防火墙规则被改动。

为什么要这样做：
- `tailscale ping` 只能证明私有网络连通。
- `Test-NetConnection` 可以进一步证明 `3306` 这个 TCP 端口可访问。

依据：
- MySQL 官方说明 TCP/IP 默认端口是 `3306`。
- Windows 防火墙官方命令 `New-NetFirewallRule` 支持按 `LocalPort`、`Protocol`、`LocalAddress`、`RemoteAddress` 创建入站规则；当前组长电脑就是只放行 Tailscale 地址访问 `3306`。
- 参考：MySQL Connection Options: https://dev.mysql.com/doc/refman/8.4/en/connection-options.html
- 参考：Microsoft New-NetFirewallRule: https://learn.microsoft.com/en-us/powershell/module/netsecurity/new-netfirewallrule

## 6. 队友第四步：拉取最新代码

打开 PowerShell，进入项目目录。如果你的项目在 `D:\whu-circle-prototype`：

```powershell
cd D:\whu-circle-prototype
git pull origin main
```

如果你的项目在其他位置，把路径换成自己的真实路径。

为什么要这样做：
- 前后端代码、SQL 脚本和 API 封装都在仓库里。
- 使用共享数据库时，大家最好使用同一版代码，否则可能出现字段不匹配或接口不一致。

依据：
- 本项目配置文件位于 `backend/src/main/resources/application-mysql.yml`，后端 MySQL 连接方式随代码仓库一起分发。

## 7. 队友第五步：安装前端依赖

在项目根目录执行：

```powershell
npm install
```

为什么要这样做：
- 前端是 React + Vite 项目。
- `npm install` 会根据 `package-lock.json` 安装前端运行需要的依赖。

依据：
- 项目根目录存在 `package.json` 和 `package-lock.json`。

## 8. 队友第六步：配置后端连接共享数据库

进入后端目录：

```powershell
cd D:\whu-circle-prototype\backend
```

在当前 PowerShell 窗口设置三个环境变量：

```powershell
$env:DB_URL="jdbc:mysql://100.97.230.36:3306/whu_circle?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai"
$env:DB_USERNAME="whu_team"
$env:DB_PASSWORD="这里填写组长单独发给你的数据库密码"
```

注意：
- `DB_PASSWORD` 不要提交到 GitHub。
- 不要把真实密码写进 README、Markdown 文档、Issue、群公告或截图。
- 这个写法只对当前 PowerShell 窗口有效；关掉窗口后需要重新设置。

为什么要这样做：
- 后端 `application-mysql.yml` 明确读取 `DB_URL`、`DB_USERNAME`、`DB_PASSWORD`。
- Spring Boot 支持从操作系统环境变量读取配置。
- 当前 PowerShell 临时环境变量不会污染仓库，也不会被 Git 提交。

依据：
- 项目文件：`backend/src/main/resources/application-mysql.yml`
- Spring Boot 官方说明支持环境变量作为外部化配置来源。
- Spring Boot 官方说明 `${name:default}` 占位符可以在配置值里使用。
- 参考：Spring Boot Externalized Configuration: https://docs.spring.io/spring-boot/reference/features/external-config.html

## 9. 队友第七步：启动后端

仍然在 `backend` 目录执行：

```powershell
mvn spring-boot:run "-Dspring-boot.run.profiles=mysql"
```

看到类似下面的信息，说明后端启动成功：

```text
Tomcat started on port 8080
Started WhuCircleApplication
```

启动后可以打开：

```text
http://127.0.0.1:8080/api/v1/health
```

如果返回健康检查结果，说明后端正在运行。

为什么要这样做：
- 不加 `mysql` profile 时，后端默认可能使用 `mock` 内存数据，不会连接共享 MySQL。
- 加上 `-Dspring-boot.run.profiles=mysql` 后，才会加载 `application-mysql.yml` 里的数据库配置。

依据：
- 项目文件：`backend/src/main/resources/application-mysql.yml` 通过 `spring.config.activate.on-profile: mysql` 声明只在 `mysql` profile 生效。
- Spring Boot 支持 profile-specific 配置文件和命令行参数。
- 参考：Spring Boot Externalized Configuration: https://docs.spring.io/spring-boot/reference/features/external-config.html

## 10. 队友第八步：启动前端

另开一个 PowerShell 窗口，进入项目根目录：

```powershell
cd D:\whu-circle-prototype
npm run dev
```

终端里会出现本机访问地址，通常是：

```text
http://127.0.0.1:5173
```

如果 `5173` 被占用，Vite 可能自动换成 `5174` 或其他端口，以终端显示为准。

为什么要这样做：
- 前端只负责页面展示和请求转发。
- 后端必须已经启动在 `8080`，前端页面里的接口请求才会正常返回。

依据：
- 项目根目录存在 `vite.config.mjs`，前端开发服务器由 Vite 启动。

打开前端地址后，可以使用演示账号：

为什么要这样做：
- 这些操作会产生真实的数据库读写。
- 如果另一个队友刷新页面后也能看到同一条笔记或评论，就说明大家正在使用同一个共享数据库。

依据：
- 当前数据库账号拥有 `SELECT`、`INSERT`、`UPDATE`、`DELETE` 权限，可以完成业务数据读写。
- 项目后端 MySQL repository 已经覆盖用户、笔记、评论、通知、频道、聊天等业务数据。

## 12. 可选：用 MySQL 客户端直接验证连接

如果电脑上安装了 MySQL 命令行客户端，可以执行：

```powershell
mysql -h 100.97.230.36 -P 3306 -u whu_team -p -D whu_circle --protocol=tcp
```

输入密码后，再执行：

```sql
SELECT DATABASE();
SELECT CURRENT_USER();
SHOW TABLES;
```

如果能看到 `whu_circle` 和表列表，说明数据库账号可用。

为什么要这样做：
- 这一步绕过 Spring Boot，直接验证 MySQL 账号、密码、端口和数据库名。
- 如果这里失败，后端启动时也大概率会连接失败。

依据：
- MySQL 官方连接选项包含 `--host`、`--port`、`--user`、`--password`、`--protocol`。
- MySQL 官方说明 `--protocol=TCP` 可以强制使用 TCP/IP 连接。
- 参考：MySQL Connection Options: https://dev.mysql.com/doc/refman/8.4/en/connection-options.html

## 13. 常见问题

### 13.1 后端启动时报 `Access denied`

可能原因：
- `DB_USERNAME` 写错。
- `DB_PASSWORD` 写错。
- 队友没有通过 Tailscale 连接进来，导致 MySQL 没有匹配到 `'whu_team'@'100.%'` 这个账号来源。

处理方式：

```powershell
tailscale ping 100.97.230.36
Test-NetConnection 100.97.230.36 -Port 3306
```

确认网络通了以后，重新设置：

```powershell
$env:DB_USERNAME="whu_team"
$env:DB_PASSWORD="组长单独发给你的密码"
```

然后重新启动后端。

### 13.2 后端启动后还是看不到共享数据

检查启动命令是不是用了 `mysql` profile：

```powershell
mvn spring-boot:run "-Dspring-boot.run.profiles=mysql"
```

如果只执行：

```powershell
mvn spring-boot:run
```

后端可能会使用默认 `mock` 数据，数据只存在内存里，刷新或重启后表现会和共享库不一致。

### 13.3 前端能打开，但页面接口报错

先确认后端健康检查：

```text
http://127.0.0.1:8080/api/v1/health
```

如果后端没有启动，前端页面能打开也无法正常使用接口。

### 13.4 想改表结构怎么办

队友账号目前不能直接 `ALTER TABLE`、`DROP TABLE` 或 `CREATE TABLE`。

正确流程：

1. 在代码里修改 SQL migration 或 schema。
2. 发给组长确认。
3. 由组长用 root 或管理账号执行结构变更。
4. 所有人拉取最新代码后继续联调。

为什么要这样做：
- 表结构变化会影响所有人。
- 业务数据可以让队友自由测试，但表结构需要集中处理，避免某个人本地操作导致整组后端都启动失败。

依据：
- MySQL `GRANT` 可以按权限类型细分；当前只授予业务数据 DML 权限，不授予 DDL 权限。
- 参考：MySQL GRANT Statement: https://dev.mysql.com/doc/refman/8.4/en/grant.html

## 14. 组长当前已经完成的主机侧配置

这一节是记录，不需要队友重复执行。

1. 组长电脑已安装并登录 Tailscale。
2. 组长电脑 Tailscale IP 是 `100.97.230.36`。
3. 组长电脑 MySQL 监听 `3306`。
4. 组长电脑 Windows 防火墙已添加规则，只允许 Tailscale 地址段访问本机 Tailscale IP 上的 `3306`。
5. 组长电脑已创建 MySQL 账号 `whu_team`，来源限制为 `'100.%'`。
6. 组长电脑已给 `whu_team` 授予 `whu_circle.*` 的 `SELECT`、`INSERT`、`UPDATE`、`DELETE` 权限。

为什么这样配置：
- Tailscale IP 稳定，适合队友长期连接。
- MySQL 只通过 Tailscale 私有网络访问，不直接暴露公网。
- 数据库账号只允许来自 `100.%` 的 Tailscale 地址，减少非 tailnet 来源访问。
- DML 权限能满足开发测试，DDL 和授权权限保留给组长。

依据：
- Tailscale 官方说明设备会获得稳定的 `100.x.y.z` 地址。
- MySQL 官方说明 `CREATE USER 'user'@'host'` 使用账号名和 host 部分限制来源。
- MySQL 官方说明 `GRANT` 可授予具体权限，`WITH GRANT OPTION` 需要谨慎。
- Windows 官方说明 `New-NetFirewallRule` 可以按本地地址、远程地址、协议和端口创建入站规则。
- 参考：How Tailscale assigns IP addresses: https://tailscale.com/docs/concepts/ip-and-dns-addresses
- 参考：MySQL CREATE USER Statement: https://dev.mysql.com/doc/refman/8.4/en/create-user.html
- 参考：MySQL GRANT Statement: https://dev.mysql.com/doc/refman/8.4/en/grant.html
- 参考：Microsoft New-NetFirewallRule: https://learn.microsoft.com/en-us/powershell/module/netsecurity/new-netfirewallrule

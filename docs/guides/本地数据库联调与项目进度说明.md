这份文档给小组成员用来了解：项目现在做到哪里了，加入 MySQL 之后启动流程为什么变复杂了，以及每个人在自己电脑上应该怎样启动和测试。

## 1. 当前项目状态

项目名称：WHU Circle（武大校园圈）

当前目标：做出一个可以在本机展示和联调的校园社交平台初步版本。

当前代码位置：

```text
D:\whu-circle-prototype
```

当前主要模块：

- 后端：Spring Boot，负责 API 接口、登录注册、业务规则。
- 数据库：MySQL，负责保存用户、笔记、频道、聊天、通知等真实数据。

现在已经接入或基本完成的功能：

- 主页公开笔记、笔记详情、评论、点赞、收藏。

- 频道列表、加入频道、频道帖子、回复、点赞、置顶、公告。

目前不建议直接修改的大块：

- 管理员功能目前还没有正式合并到远程主分支，后续应单独开发和测试。

## 2. SQL 和 MySQL 是什么

SQL 是一种操作数据库的语言。我们可以用 SQL 创建表、插入数据、查询数据、修改数据。

MySQL 是一个数据库软件。它负责真正保存项目数据。

简单理解：

```text
前端页面
  ↓ 发请求
后端 API
  ↓ 查询或写入
MySQL 数据库
```

例如用户登录时：

1. 前端把邮箱和密码发给后端。
2. 后端去 MySQL 的 `users` 表里查这个邮箱。
3. 如果查到用户，再校验密码。
4. 校验成功后，后端返回登录 token。
5. 前端保存 token，后续请求就能带着登录状态。

所以加入 MySQL 后，后端不再只靠代码里的 mock 假数据，而是要连接一个真正运行中的数据库。

## 3. 加入数据库后有什么影响

以前如果只看静态前端，启动前端就能看到页面。

现在如果要测试真实流程，需要同时满足三件事：

```text
MySQL 已启动
后端已启动
前端已启动
```

缺任何一个都会出问题：

- MySQL 没启动：后端会启动失败，常见错误是 `Can't connect to MySQL server on 127.0.0.1:3306`。
- 后端没启动：前端页面可以打开，但登录、笔记、频道等接口会失败。
- 前端没启动：浏览器打不开页面。

因此推荐启动顺序：

```text
先启动 MySQL
再启动后端
最后启动前端
```

## 4. 为什么每个人都要本机建数据库

我们现在做的是本地项目展示，不是线上服务器项目。

也就是说，每个人电脑上的后端默认连接的是：

```text
127.0.0.1:3306
```

`127.0.0.1` 的意思是“我自己的电脑”。

所以 A 同学电脑上的 `127.0.0.1` 指 A 的电脑，B 同学电脑上的 `127.0.0.1` 指 B 的电脑。它不会自动连接到组长电脑。

因此每个人如果要在自己电脑上完整跑通后端，就需要：

- 自己安装或启动 MySQL。
- 自己初始化 `whu_circle` 数据库。
- 自己配置本机数据库密码。

这样做的好处：

- 每个人都能独立调试，不互相影响。
- 不会因为一个人的电脑关机导致全组都连不上数据库。
- 测试数据可以随时重建。

## 6. 每个人第一次拉取项目后要做什么

### 6.1 拉取代码

```powershell
cd D:\
git clone https://github.com/oliviawxx/whucircle.git whu-circle-prototype
cd D:\whu-circle-prototype
```

如果已经 clone 过：

```powershell
cd D:\whu-circle-prototype
git pull
```

### 6.2 安装前端依赖

```powershell
cd D:\whu-circle-prototype
npm install
```

### 6.3 准备 MySQL

先确认 MySQL 服务是否启动：

```powershell
Get-Service MySQL,MySQL80
```

如果显示 `Stopped`，尝试用管理员 PowerShell 启动：

```powershell
Start-Service MySQL
```

如果你电脑上服务名是 `MySQL80`：

```powershell
Start-Service MySQL80
```

检查 3306 端口是否监听：

```powershell
netstat -ano | findstr 3306
```

看到 `LISTENING` 才说明 MySQL 服务真的起来了。

如果服务启动不了，也可以临时启动 MySQL：

```powershell
cd D:\sql\mysql-9.7.1-winx64
.\bin\mysqld.exe --basedir=D:\sql\mysql-9.7.1-winx64 --datadir=D:\sql\mysql-9.7.1-winx64\data --port=3306 --console
```

这个窗口不要关。另开一个 PowerShell 再继续操作。

## 7. 初始化数据库

如果 MySQL 已加入 PATH，可以这样执行：

```powershell
cd D:\whu-circle-prototype\backend
$env:MYSQL_PWD="你的 root 密码"
cmd /c "mysql -u root -h 127.0.0.1 -P 3306 --protocol=tcp --default-character-set=utf8mb4 < sql\001_schema.sql"
cmd /c "mysql -u root -h 127.0.0.1 -P 3306 --protocol=tcp --default-character-set=utf8mb4 < sql\002_seed.sql"
Remove-Item Env:MYSQL_PWD
```

如果 MySQL 没有加入 PATH，用完整路径：

```powershell
cd D:\whu-circle-prototype\backend
$env:MYSQL_PWD="你的 root 密码"
cmd /c ""D:\sql\mysql-9.7.1-winx64\bin\mysql.exe" -u root -h 127.0.0.1 -P 3306 --protocol=tcp --default-character-set=utf8mb4 < "D:\whu-circle-prototype\backend\sql\001_schema.sql""
cmd /c ""D:\sql\mysql-9.7.1-winx64\bin\mysql.exe" -u root -h 127.0.0.1 -P 3306 --protocol=tcp --default-character-set=utf8mb4 < "D:\whu-circle-prototype\backend\sql\002_seed.sql""
Remove-Item Env:MYSQL_PWD
```

说明：

- 这里用 `cmd /c ... < sql文件` 是为了让 MySQL 直接读取 UTF-8 SQL 文件。
- 不建议用 PowerShell 管道导入中文 SQL，容易出现中文乱码。

初始化后可以检查：

```powershell
& "D:\sql\mysql-9.7.1-winx64\bin\mysql.exe" -u root -p -h 127.0.0.1 -P 3306
```

进入 MySQL 后执行：

```sql
SHOW DATABASES;
USE whu_circle;
SHOW TABLES;
SELECT id, email, nickname FROM users LIMIT 10;
```

能看到 `whu_circle`、表名和演示用户，就说明数据库初始化成功。

## 8. 后端配置方式

后端使用 Spring Boot。

默认情况下，如果直接运行：

```powershell
mvn spring-boot:run
```

后端会使用 mock 模式，主要用于快速启动，不连接真实 MySQL。

如果要连接 MySQL，需要使用 `mysql` profile：

```powershell
mvn spring-boot:run "-Dspring-boot.run.profiles=mysql"
```

后端 MySQL 配置在：

```text
backend/src/main/resources/application-mysql.yml
```

核心配置是：

```yaml
spring:
  datasource:
    url: jdbc:mysql://127.0.0.1:3306/whu_circle
    username: whu_circle
    password: 从环境变量 DB_PASSWORD 读取
```

所以后端连接数据库时，需要知道：

- 数据库地址：`127.0.0.1`
- 数据库端口：`3306`
- 数据库名：`whu_circle`
- 数据库账号：`whu_circle`
- 数据库密码：每个人本机自己设置

推荐在 `backend/.env` 中保存本机密码，但 `.env` 不要提交到 Git。

也可以启动前临时设置：

```powershell
$env:DB_PASSWORD="你的数据库密码"
```

如果本机账号不是 `whu_circle`，也可以设置：

```powershell
$env:DB_USERNAME="你的数据库账号"
$env:DB_PASSWORD="你的数据库密码"
```

前端地址：

## 10. 前端和后端是怎么连起来的

前端代码里请求的是：

```text
/api/v1/...
```

不是直接写：

```text
http://127.0.0.1:8080/api/v1/...
```

原因是 Vite 配置了代理：

```js
server: {
  proxy: {
    "/api": {
      target: "http://127.0.0.1:8080"
    }
  }
}
```

所以浏览器访问前端时：

```text
http://127.0.0.1:5173/api/v1/notes
```

Vite 会自动转发到：

```text
http://127.0.0.1:8080/api/v1/notes
```

这就是为什么前端和后端要同时启动。

## 12. 常见错误

### 12.1 `Can't connect to MySQL server on 127.0.0.1:3306`

含义：MySQL 客户端或后端找不到正在运行的 MySQL 服务。

解决：

```powershell
Get-Service MySQL,MySQL80
netstat -ano | findstr 3306
```

如果没有 `LISTENING`，先启动 MySQL。

### 12.2 `Start-Service MySQL` 启动失败

可能原因：

- PowerShell 不是管理员身份。
- MySQL 服务配置坏了。
- MySQL 数据目录有问题。
- 端口被占用。

先用管理员 PowerShell 试：

```powershell
Start-Service MySQL
```

如果还不行，用 `mysqld.exe --console` 临时启动，看控制台具体错误。

### 12.3 后端 `BUILD FAILURE`

如果日志里有 MySQL 连接错误，本质通常不是 Java 编译失败，而是后端启动时连不上数据库。

先检查：

```powershell
netstat -ano | findstr 3306
```

再确认数据库密码和 `DB_PASSWORD`。

### 12.4 `No plugin found for prefix spring-boot`

通常是因为不在后端目录执行 Maven。

错误位置：

```powershell
PS C:\Users\xxx> mvn spring-boot:run
```

正确位置：

```powershell
cd D:\whu-circle-prototype\backend
mvn spring-boot:run "-Dspring-boot.run.profiles=mysql"
```

### 12.5 页面打开了，但登录/笔记/频道失败

通常是后端没开，或者后端没连上数据库。

检查：

```text
http://127.0.0.1:8080/api/v1/health
```

如果这个地址打不开，说明后端没启动成功。

## 13. 终止项目

如果是在终端里启动的，直接在对应窗口按：

```text
Ctrl+C
```

如果忘了哪个窗口在运行，可以按端口结束：

```powershell
$ports = @(5173, 8080)
$listeners = Get-NetTCPConnection -State Listen -ErrorAction SilentlyContinue | Where-Object { $ports -contains $_.LocalPort }
$processIds = $listeners | Select-Object -ExpandProperty OwningProcess -Unique
foreach ($processId in $processIds) { Stop-Process -Id $processId -Force }
```

注意：这只会关前端和后端，不会关 MySQL。

如果是用 `mysqld.exe --console` 临时启动的 MySQL，关闭那个 MySQL 窗口或按 `Ctrl+C` 即可停止。

## 14. 协作建议

- 前端同学主要关注 `src` 目录。
- 后端同学主要关注 `backend/src/main/java`。
- 数据库同学主要关注 `backend/sql`。
- 修改接口时，同步更新 `docs/api/WHU_Circle_后端API开发清单.md`。
- 修改数据库表结构时，同步更新 `backend/sql/001_schema.sql`。
- 修改演示数据时，同步更新 `backend/sql/002_seed.sql`。
- 不要提交本机密码、`.env`、邮箱授权码。

如果某个功能在自己电脑上无法复现，先确认三件事：

```text
MySQL 是否启动
后端是否启动
前端是否启动
```

这三个确认之后，再看具体接口或页面逻辑。

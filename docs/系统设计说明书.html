1. 系统体系架构

1.1 总体架构

前后端分离四层结构：浏览器 → React + Vite 前端 → Spring Boot 后端 → MySQL（业务数据）+ MinIO（图片文件）。团队联调通过 Tailscale 内网穿透共享组长电脑的 MySQL 和 MinIO。

flowchart LR
    B["浏览器"] --> F["React+Vite :5173"]
    F -->|"/api/v1"| B2["SpringBoot :8080"]
    F -->|"图片"| M["MinIO :9000"]
    B2 --> S["Service"] --> R["Repository"]
    R --> D["MySQL"] & K["Mock内存"]
    B2 --> M & L["uploads/images/"]

1.2 前端架构

单页应用，App.jsx 集中管理登录态、导航和弹窗状态。src/api/client.js 统一 Bearer Token 与错误处理。

flowchart TD
    A["App.jsx"] --> U["AuthPage"] & L2["Sidebar+Topbar"] & MD["AppModals 弹窗"]
    A --> P["pages/"]
    P --> H["HomePage"] & SC["SocialCircle"] & CH["Channels"] & CT["Chats"]
    P --> PF["Profile"] & SV["Saved"] & ST["Settings"] & AD["Admin"]
    A --> API["src/api/"] --> CL["client.js"]

1.3 后端架构

Controller → Service → Repository 三层分层，AuthenticationInterceptor 拦截认证。12 个 Controller、11 个 Service、7 个 Repository 接口，MySQL 和 Mock 双实现。

flowchart TD
    C["Controller x12"] --> S3["Service x11"] --> R2["Repository x7"]
    R2 --> RM["mysql/"] & RI["mock/"]
    C --> DTO["DTO"] & DOM["Domain record x11"]
    I["AuthInterceptor"] --> T["TokenService"]
    RC["RecommendationService"] --> JD["JdbcClient 反馈持久化"]

1.4 部署架构

4. 复杂功能算法设计

4.1 笔记可见性判断

flowchart TD
    A["canView(uid,note)"] --> B{"authorId==uid?"}
    B -->|Y| OK["允许"]
    B -->|N| C{"拉黑?"}
    C -->|Y| NO["拒绝"]
    C -->|N| D{"visibility?"}
    D -->|PUBLIC| OK
    D -->|PRIVATE| NO
    D -->|FRIENDS| E{"好友?"}
    E -->|Y| OK
    E -->|N| NO

function canView(userId, note):
  if note.authorId == userId: return true
  if isBlockedEitherWay(userId, note.authorId): return false
  if note.visibility == PUBLIC: return true
  if note.visibility == FRIENDS:
    return relation(userId, note.authorId) == FRIEND
  return false

4.2 社交圈可见性判断

在 canView 基础上增加关注关系限定，用于社交圈笔记流：

function isSocialVisible(userId, note):
  if note.authorId == userId: return false
  if !canView(userId, note): return false
  if note.visibility == FRIENDS: return true
  if note.visibility == PUBLIC:
    rel = relation(userId, note.authorId)
    return rel == FOLLOWING or rel == FRIEND
  return false

4.3 个性化推荐评分算法

对笔记、用户、频道三类目标独立评分，首页混合各取 4 条按评分降序展示。评分受隐私开关控制。

笔记推荐评分（多因子加权）：

function scoreNote(currentUser, note):
  score = 35
  if sameCollege: score += 18
  if sameGrade: score += 8
  if relation == FRIEND: score += 22
  if relation == FOLLOWING: score += 15
  overlap = countIntersection(note.tags, interestTags)
  score += overlap * 8
  score += min(likeCount*0.7 + commentCount*0.9, 18)
  return score

用户推荐评分：基础 30 + 同学院 20 + 同年级 12 + 对方已关注你 14 + min(粉丝×0.4+关注×0.2, 10)。关个性化推荐时不启用学院/年级维度。

频道推荐评分：基础 28 + min(成员数×0.25, 18) + 公开频道 10 + 好友关系密度×14。关闭个性化推荐时仅保留公开/成员数维度。

4.6 关系状态计算

function relation(uid, tid):
  if isBlockedEitherWay(uid, tid): return BLOCKED
  i = follows.exists(uid, tid)
  m = follows.exists(tid, uid)
  if i && m: return FRIEND
  if i: return FOLLOWING
  if m: return FOLLOWER
  return NONE

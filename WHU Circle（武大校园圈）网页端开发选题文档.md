# WHU Circle（武大校园圈）网页端开发选题文档

## 1. 选题基本信息

### 1.1 选题名称

WHU Circle（武大校园圈）网页端开发选题文档

### 1.2 项目背景、定位与目标

#### 1.2.1 项目背景

在大学生活中，学生的信息交流往往分散在班级群、社团群、朋友圈、论坛和各类通知渠道中，内容容易被刷屏覆盖，也缺少统一的沉淀和检索入口。学习经验分享、活动信息发布、兴趣交流、二手信息和日常求助等内容，都需要一个更集中、更轻量的交流平台承载。

WHU Circle（武大校园圈）拟以网页端形式构建一个面向学生群体的社交媒体平台，帮助用户在同一平台内完成内容发布、好友互动、社交圈管理和信息发现。

#### 1.2.2 平台定位与作用

本平台定位为校园场景下的轻量级社交媒体系统，主要承载以下作用：

- 作为学生发布动态、分享生活和表达观点的内容空间。
- 作为用户发现校园信息、活动内容和兴趣话题的信息入口。
- 作为好友交流、私信沟通和社交圈管理的互动平台。
- 作为平台方进行用户管理、内容管理和权限控制的基础系统。

用户注册采用武大校内邮箱验证，用于确认用户身份来源，提升平台内容和互动关系的可信度。

#### 1.2.3 使用场景

平台主要适用于以下场景：

- 学生发布学习记录、生活动态、图片内容和观点分享。
- 用户浏览动态广场，发现感兴趣的内容和用户。
- 用户通过点赞、评论、私信和群聊进行互动。
- 用户按照好友、同学、社团或兴趣方向管理自己的社交圈。
- 用户通过推荐和发现功能获取更符合兴趣的内容。
- 用户通过隐私设置控制个人资料、动态内容和互动权限。

#### 1.2.4 项目目标

本项目目标是完成一个结构清晰、功能闭环、可演示的基础版社交媒体平台原型。系统应覆盖校内邮箱注册验证、用户资料、多媒体动态、好友关系、私信群聊、推荐发现和隐私控制等核心流程。项目不追求工业级大规模并发和复杂算法效果，重点保证基础功能完整、页面交互清晰、数据流程可靠。

### 1.3 技术路线

前端技术：

- Vue 3
- HTML、CSS、JavaScript
- Element Plus 组件库
- Axios 网络请求库

后端技术：

- Spring Boot
- MyBatis
- MySQL
- JWT 登录认证

其他工具：

- Git + GitHub 版本控制与代码托管，项目已建立 GitHub 仓库
- Apifox / Postman 接口测试
- Navicat / DataGrip 数据库管理

### 1.4 系统功能设计

#### 1.4.1 基础功能

基础功能是项目优先完成的核心内容，用于保证系统形成完整的武大校内社交流程。

- 武大校内邮箱注册验证
- 用户登录、退出登录
- 用户个人资料创建和编辑
- 头像上传与个人主页展示
- 动态发布、删除和查看
- 图片等多媒体内容分享
- 动态广场展示与内容发现
- 点赞与取消点赞
- 评论与删除评论
- 好友添加、删除和好友列表管理
- 社交圈分组与基础管理
- 私信和群聊
- 个性化内容推荐和发现
- 隐私设置和安全控制

#### 1.4.2 拓展功能

拓展功能是在基础功能完成后，根据开发时间和实现情况选择增加的内容，主要参考真实社交媒体产品的常见能力。

- 校园话题广场与话题标签
- 校园热榜和趋势榜单
- 消息通知中心
- 内容举报与审核后台
- 社团、班级或活动主页
- 校园活动发布与报名
- 失物招领、二手交易等校园生活频道
- 更细化的推荐算法和内容排序

### 1.5 项目难点

本项目的主要难点包括：

- 校内邮箱验证和用户身份认证。
- 用户登录状态和权限控制。
- 动态、评论、点赞、好友、群聊之间的数据关系设计。
- 防止重复点赞等异常操作。
- 图片上传、存储和展示。
- 动态列表的分页查询和排序。
- 私信、群聊和隐私权限的基础控制。
- 用户只能删除自己的动态或评论。

### 1.6 预期成果

项目完成后，应形成一个可运行的基础版武大校内社交媒体平台，能够完成用户从校内邮箱注册验证、登录、创建资料、发布动态、浏览推荐内容、点赞评论、管理好友、私信群聊到设置隐私权限的完整流程。同时，应提供项目文档、数据库设计说明、接口说明和测试用例。

### 2.2 成员1：前端开发负责人

主要任务：

- 设计并实现系统主要网页页面。
- 完成登录注册、动态广场、个人主页、好友列表、消息页面等前端页面。
- 实现发布动态、点赞、评论、好友管理、私信群聊等交互效果。

- 完成前后端数据联调。
- 完成最终项目演示界面。

### 2.3 成员2：后端开发负责人

主要任务：

- 搭建后端项目结构。
- 实现用户注册、登录和身份认证。
- 实现用户资料、动态、评论、点赞、好友、私信群聊等核心接口。
- 处理用户权限校验和异常提示。
- 提供头像和动态图片上传接口。

阶段成果：

- 完成后端核心接口。
- 完成用户、动态、评论、点赞、好友和消息等业务逻辑。
- 配合前端完成接口联调。

### 2.4 成员3：数据库与测试负责人

主要任务：

- 设计系统数据库表结构。
- 建立用户、动态、图片、评论、点赞、好友、消息、群聊等核心数据表。
- 准备测试数据并进行功能测试。
- 记录接口测试结果，整理项目文档。

阶段成果：

- 完成数据库设计和测试数据。
- 完成核心功能测试记录。
- 完成项目文档和演示材料。

### 2.5 协作方式

项目采用 Git 进行版本控制，并已建立 GitHub 仓库用于代码托管、版本管理和成员协作。三名成员分别负责前端、后端、数据库测试与文档，但在接口设计、功能联调和最终演示阶段需要共同协作。

建议协作流程：

1. 共同确定系统功能范围和数据库表结构。
2. 后端先提供接口文档和模拟数据。
3. 前端根据接口文档完成页面开发。
4. 数据库与测试负责人准备测试数据并进行接口测试。
5. 三人共同完成前后端联调、问题修复和项目演示。

## 3. 初版总结

WHU Circle（武大校园圈）是一个面向校园交流场景的基础版社交媒体平台。平台主要承载学生内容分享、信息发现、好友互动和社交圈管理等需求，适用于学习生活分享、活动交流、兴趣讨论和日常沟通等场景。

本项目适合三人小组进行网页端开发，功能覆盖邮箱认证、用户资料、多媒体动态、好友关系、私信群聊、推荐发现和隐私控制等核心能力。开发过程中应优先保证基础流程完整稳定，再根据时间增加话题广场、校园热榜、通知中心、内容审核、活动主页等更接近真实竞品的拓展功能。
